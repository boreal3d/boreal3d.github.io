Welcome to Boreal3D

A synthetic 3D forest point cloud dataset, specifically developed to support advanced research in large-scale forest scene understanding and structural parameter estimation.

Overview

Boreal3D is a comprehensive synthetic forest point cloud dataset designed to support advanced research in large-scale forest scene understanding and fine-grained 3D forest structure analysis. It includes 1,000 highly realistic and structurally diverse forest plots, collected across four different platforms. The dataset contains 48,403 trees and over 35.3 billion data points, offering an invaluable resource for ecological research, deep learning applications, and the detailed analysis of forest structures.

Each point in the dataset is meticulously labeled with semantic, instance, and viewpoint information, as well as other relevant annotations, facilitating a wide range of research applications, including but not limited to forest resource management, ecological modeling, and environmental monitoring. Additionally, each tree is characterized by key structural parameters, including tree diameter, crown width, leaf area, and wood volume, making it an essential tool for large-scale forest scene understanding, fine-grained structural analysis, and forest dynamics modeling, among other applications.

Tutorial

All plot data are stored in individual folders, with folder names following the format: < PlotType > Plot < PlotNumber >. For example, EasyPlot1 represents the first plot of the "Easy" type.

Each folder contains four files: ALS.laz, ULS.laz, MLS.laz, and TLS.laz, corresponding to four different data collection platforms.

Each .laz file contains the following key fields:
  1. x,y,z: Represent the spatial coordinates of the points.
  2. semantic_id: Represents the semantic label. 0 indicates understory, 1 indicates terrain, 2 indicates Leaf, and 3 indicates Wood.
  3. instance_id: Represents the instance label. 0 indicates non-tree objects, while values from 1 to N indicate individual tree identifiers.
  4. From: Represents the viewpoint information.
  5. species_id: Represents the species label. 0 indicates non-tree objects, 1 indicates Spruce, 2 indicates Pine, and 3 indicates Birch.
